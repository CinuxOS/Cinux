# M5: ext4 基础

> 在现有 ext2 代码基础上增加 ext4 extent 支持。
> ext4 的 extent tree 替代 ext2 的间接块映射，提升大文件性能。

## 目标

支持 ext4 文件系统的 extent-based 分配，能读取 ext4 格式的磁盘。

## 任务清单

**文件**: `kernel/fs/ext2.hpp`（扩展为兼容 ext4）

- [ ] 检测 s_feature_incompat 中的 EXT4_FEATURE_INCOMPAT_EXTENTS (0x40)
- [ ] 如果有 extent flag → 使用 extent 读取路径
- [ ] 如果没有 → 保持现有间接块路径（ext2 兼容）

### T2: Extent Tree 解析

```cpp
struct ExtentHeader {
    uint16_t eh_magic;       // 0xF30A
    uint16_t eh_entries;     // 条目数
    uint16_t eh_max;         // 最大条目数
    uint16_t eh_depth;       // 树深度（0 = 叶子）
    uint32_t eh_generation;  // 生成号
};

struct ExtentEntry {
    uint32_t ee_block;       // 逻辑块号
    uint16_t ee_len;         // 长度（块数）
    uint16_t ee_start_hi;    // 物理块号高 16 位
    uint32_t ee_start_lo;    // 物理块号低 32 位
};

struct ExtentIndex {
    uint32_t ei_block;       // 索引逻辑块号
    uint32_t ei_leaf_lo;     // 子节点块号低 32 位
    uint16_t ei_leaf_hi;     // 子节点块号高 16 位
    uint16_t ei_unused;
};
```

- [ ] ExtentHeader / ExtentEntry / ExtentIndex 结构体
- [ ] extent_tree_lookup(inode, logical_block) → physical_block
- [ ] 递归遍历 extent tree（depth > 0 时）
- [ ] 处理初始化 extent（ee_len > 32768 表示未初始化）

### T3: ext4 读写路径

- [ ] 读取：extent_tree_lookup → IBlockDevice.read_blocks
- [ ] 写入：暂不支持（ext4 写入需要 extent 分配，复杂度高）
- [ ] 只读 ext4 + 写入回退到 ext2 间接块
