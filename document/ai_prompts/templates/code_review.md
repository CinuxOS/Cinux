# 审查任务

请按以下维度逐一审查，**每条建议必须给出修改前后的对比代码**：

## 1. 正确性问题（P0 - 必须修复）
- 未定义行为（UB）
- 内存安全问题（越界、use-after-free、堆栈损坏）
- x86-64 硬件行为不符（寄存器使用错误、对齐问题、原子性问题）
- AT&T 汇编语法错误或语义错误

## 2. 健壮性问题（P1 - 强烈建议修复）
- 缺少必要的边界检查
- 中断安全性（是否需要 `cli`/`sti` 保护）
- 未处理的硬件错误状态

## 3. 现代 C++ 改进（P2 - 建议）
- 可用 `constexpr` / `consteval` 替代运行时计算的地方
- 可用 `concepts` 增强模板约束的地方
- RAII 机会（资源未被正确包装）
- 可用 `[[nodiscard]]`、`[[likely]]` 等属性的地方

## 4. 可读性与可维护性（P3 - 可选）
- 命名不清晰的变量/函数
- 缺少注释的关键逻辑
- 魔法数字应提取为 `constexpr` 常量

## 5. 性能（P4 - 仅当明显时提出）
- 热路径中的不必要操作
- 缓存友好性问题
