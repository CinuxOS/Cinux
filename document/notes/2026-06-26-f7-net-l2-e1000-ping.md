# F7 网络协议栈 L2 — e1000 接入，真 ping 10.0.2.2 干通

**日期**: 2026-06-26　**分支**: `worktree-f7-net-ping`　**提交**: `92b82fd`(L2 代码) + L3 docs/脚本
**验证**: run-kernel-test **947/0**(+1) + host net 5/5 + `check_net_decoupling` 绿

## 背景
L0 抽象 + L1 loopback 端到端证明后([L0](./2026-06-26-f7-net-l0-netdev-abstraction.md)/[L1](./2026-06-26-f7-net-l1-arp-ipv4-icmp-loopback.md) note),L2 接真网卡 e1000。
**底子优先的回报**:栈在 loopback 上已对,e1000 接入**一次过**——失败会锁死在 adapter,不变成"栈还是时序"的玄学。

## 设计
- **E1000NetDevice adapter**(`drivers/net/e1000_net_device.hpp`,header-only):
  - **copy RX**:`poll_rx` 把 DMA buffer copy 进 adapter 内部 `rx_scratch_[]`,`sink=null`(recycle no-op,无需 BufferSink)。
  - `send_l3` 组 `{EthHdr || L3}` 进 `tx_buf_[]` → `ctrl_.send_packet`(e1000 再 copy 进自己的 TX DMA ring,故 `tx_buf_` 不必 DMA-able)。
  - **构造注入** `E1000Controller&`(**非 `::instance()`**——双网卡 adapter 共享同一 ring 会撞;grep 强制)。
  - **RX scratch 与 TX buf 分离**:handler 在 `on_frame` 里 `send_l3` 发 reply 不会打花在途的 RX 帧。
- **生产 `e1000_init` 补 `start_tx`**:原 `init()` 只 arm RX;L3 栈要发 ARP reply + ICMP echo,**没 TX 则 `send_l3` 组好帧却不上线(SLIRP 永不答,静默)**。
- **真 ping 内核测**(`test_net::test_ping_e1000`):e1000 + adapter + 全栈,ping `10.0.2.2`:
  - 循环:`send_echo_request`(iter1 触发 ARP resolve miss→发 ARP request;iter2+ ARP 命中→发 ICMP echo)+ `poll`(sti/hlt 让 QEMU main loop 跑 + LAPIC timer 唤醒,SLIRP reply 落 ring)。
  - 复用 **F5-M6 批b-fix 时序**(LAPIC timer + sti/hlt),**绝不写 trap 循环 hack**。

## 铁证
```
[net] loopback ping:    reply id=0xabcd seq=1 (round-trip in one poll)
[net] e1000 ping 10.0.2.2: reply id=0x1234 seq=1 (1 replies)
```
ARP resolve + ICMP echo 真往返 QEMU SLIRP 网关 10.0.2.2。

## 解耦落地(4 grep 机器执行,L3)
`scripts/check_net_decoupling.sh` + CMake target `check_net_decoupling`(`cmake --build build --target check_net_decoupling`):
1. `kernel/net/` 不得 `#include` e1000 / dma_buffer / irq。
2. NIC adapter(`*_net_device.*`)不得**代码调用** `::instance()`(注释提及 OK,grep 过滤注释行)。

**负测验证**:临时往 `net_types.hpp` 注入 `#include "kernel/arch/x86_64/irq.hpp"` → target 立刻 `FAIL: kernel/net/ pulls in a driver/DMA/arch-irq header` → `DECOUPLING VIOLATED`;恢复后复绿。**真能抓人**,不是嘴上。

## 陷阱
- **生产必须 arm TX**(上面 start_tx)——L1 的 loopback 不需要(软件设备自带收发),e1000 是真硬件必须显式 arm。
- 无新时序陷阱——正是 loopback 先行把栈逻辑与 SLIRP 时序解耦的回报。
