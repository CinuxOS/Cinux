# F3-M2 批4 — 线程组 + clone 系统调用核心

> 2026-06-18。F3-M2 批4：`sys_clone`(56) Linux 风格线程原语 + 线程组（tgid/getpid）+ CLONE_* 资源共享 + **子进程用户栈返回**（GOTCHA#18）。1 批，801→809（+8 测试），fresh 809/0。批5 cleartid/libc/闭环消费。

## 背景

批3 把 sig_actions/cwd/fd_table 改 refcount 共享对象后，clone 可按 flag 真共享。批4 实现 `sys_clone`：按 CLONE_VM/FILES/SIGHAND/FS/THREAD/SETTLS/SETTID/CLEARTID 共享或复制资源，线程组（CLONE_THREAD 同 tgid），TLS（SETTLS 设 fs_base），以及最硬的——**子进程以调用者提供的栈返回用户态**。

## 设计

- **Task 线程组字段**：`tgid`（= 组长 pid，getpid 返它）、`group_leader`、`clear_child_tid`/`set_child_tid`（CLEARTID/SETTID）。build() 初始化（核线程 tgid=0），fork 设 tgid=child_pid（新进程自身组长），clone(CLONE_THREAD) 设 tgid=父 tgid。
- **clone()（fork.cpp）**：镜像 fork 的「new Task + memcpy + 拷核栈 + ctx + fork_child_trampoline(rax=0)」，按 flag 决定 share-or-copy：CLONE_VM→共享 addr_space 指针、CLONE_FILES→共享 fd_table(acquire)、CLONE_SIGHAND→共享 sig_actions(acquire)、CLONE_FS→共享 cwd(acquire)；不带 flag 则 create_copy（fork 语义）。CLONE_THREAD→兄弟（ppid=父 ppid，不入 caller children）。
- **TLS**：CLONE_SETTLS → `child->ctx.fs_base = tls`（context_switch 切入时恢复）。
- **TID flags**：CLONE_PARENT_SETTID/CHILD_SETTID → 写 child_pid 到 ctid/ptid（CLONE_VM 共享内存，父写可见）；CLONE_CHILD_CLEARTID → 记 clear_child_tid（批5 exit 用）。

## 关键陷阱

### 1. GOTCHA#18：clone 子进程用户栈返回——patch 帧 user_rsp 槽

fork 子进程返回父栈（CoW 共享），clone 子进程要返回**调用者提供的 `stack`**。crux：syscall_entry 从 `%gs:0`(=kernel_stack_top) 载核栈，建的 pt_regs 帧（96B，12 槽）固定在 `[kernel_stack_top-96, kernel_stack_top)`，**user_rsp 在帧 offset 0 = `kernel_stack_top-96`**。

clone 复用 fork 的「拷父核栈」机制（子核栈是父栈的副本，含 syscall 帧），然后 **patch 子帧 user_rsp 槽**：
```cpp
if (stack != 0)
    *(uint64_t*)(child->kernel_stack_top - 96) = stack;
```
子进程经 fork_child_trampoline(rax=0) 解卷回 syscall_entry 岭 → SYSRET 时 user_rsp=stack、user_rip=父的岭（CLONE_VM 共享代码）、rax=0。**无需从 current_rsp 算偏移**——帧在栈顶固定位置，直接按 kernel_stack_top 定位。备选 Approach C（独立 trampoline 直跳用户态）未用，patch 更简。

### 2. 调试 saga：getpid→tgid 暴露的 TEST_ASSERT 早返回悬垂（非踩踏，是悬垂指针）

批4 改 `sys_getpid` 返 `tgid` 后，整套挂死（timeout 40 救场），崩点显示在 `test_sys_open::test_open_valid_path` → sys_open → FDTable::alloc → Spinlock::acquire，崩溃 task `tid=0 pid=646 name=''` 全垃圾，诊断打印 `current_fd_table: fd_table=0x0AFFFFFF81072946`（垃圾）。**极具迷惑性**——像是内存踩踏。

真因：getpid 返 tgid 使既有 `test_sys_getpid_returns_nonnegative` 断言失败（测试设 `tmp.pid=42` 但 `tmp.tgid=0` → getpid 返 0 ≠ 42）。**`TEST_ASSERT` 失败时 `return`（早返回）→ 跳过函数末尾的 `set_current(prev)` → `Scheduler::current_` 悬垂指向已销毁的栈 `Task tmp`** → 后续测试读悬垂 current → fd_table 垃圾 → 崩。**不是踩踏，是悬垂指针**；getpid→tgid 只是触发器（改了返回值使断言由过变挂）。

诊断关键：二分（git stash 批4 → 批3 过；逐项回退 → 锁定 getpid）；加诊断 kprintf 打 current_fd_table 的 task/fd_table 地址 → fd_table 垃圾 → current_ 悬垂。修：测试设 `tmp.tgid = tmp.pid`（断言过 → set_current 执行 → 不悬垂）。

**通用教训**：改公共接口（getpid 返回值）后，依赖它的测试断言可能由过变挂；而 `TEST_ASSERT` 早返回若跳过了清理（set_current 还原），留下悬垂状态污染后续测试——表象（远处的 Spinlock 崩）远离真因（早返回的 set_current）。改接口后 grep 所有用到该返回值的测试断言同步改。

### 3. test_fork_exec 的 latent 栈下溢 + fork/clone full_used 上限

`test_dispatch_sys_fork` 设 `tmp.kernel_stack_top = rsp + 16384`（= 栈大小），致 fork 的 `full_used = kernel_stack_top - current_rsp ≈ 17KB > 16KB 栈` → `child_stack_start` 下溢到栈映射前 → memcpy 踩邻接。批3 latent（侥幸未命中要害），批4 Task 变大改堆布局后命中。修：test 改 `rsp+4096`（full_used < 16KB）+ remove_task 子进程（防被调度跑 trampoline 解卷测试栈）+ InterruptGuard。fork/clone 加 `if (full_stack_used > stack_size) full_stack_used = stack_size;` 防御（生产永不触发，栈远未满）。

### 4. clone 测试须防子进程被调度

clone 测试用栈 `Task tmp` 作 current（同 test_fork_exec 范式），clone 子进程入 run-queue。若被调度 → 跑 fork_child_trampoline → 解卷拷来的测试栈 → SYSRET 进垃圾 → 崩。修：clone+inspect+remove_task 包在 InterruptGuard（IF=0 防 tick 抢占），remove_task 出队子进程。
