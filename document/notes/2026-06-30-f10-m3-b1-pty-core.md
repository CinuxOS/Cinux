# F10-M3 Phase 2 批1 — PTY 核心纯逻辑 + host 单测

> 里程碑:F10-M3 TTY Phase 2(PTY)。批1。分支 `feat/f10-m3-pty`。commit `bab9caf`。
> 前置:Phase 1(PR#46,`TTY` 行规范 + ConsoleTty)+ DevFS(PR#48)+ session/pgrp(F3-M3)全在 main。

## 背景

Phase 1 把 console TTY 当单例做完(行规范 / 阻塞读 / ioctl TCGETS 等 / 信号),但显式把
PTY master/slave、`/dev/ptmx`、`/dev/pts/N`、`TIOCSCTTY` 推迟到「DevFS 之后」——因为 PTY 节点
必须是设备 inode。DevFS(PR#48)合入后这条路解锁。批1 只做**纯逻辑的 PTY 对**:不碰内核装配、
不碰 DevFS,目标是把"master 写 → slave 行规范 → slave 读;slave 写 → master 读"这条数据通路
在 host 单测里站住,后续批(B2-B3)再把它接到 fd/InodeOps/DevFS。

## 目标

- 一个 `Pty` 类:master 端 + slave 端,slave 持一个复用的 `TTY` 行规范。
- 四条数据路径全通(见下"设计")。
- 纯逻辑:零 kprintf / 零 proc / 零 Console,能编链进 host 单测(同 `tty.cpp` 范式)。
- 阻塞(等数据)是调度问题,留 B3 kernel 侧;批1 的原语全部非阻塞,返回搬动的字节数。

## 设计

数据流(详见 `pty.hpp` 文件头注释):

| 路径 | 实现 |
|------|------|
| `master_write(buf,n)` | 逐字节 `slave_tty_.input_char(c)`(canonical 编辑 / 回显 / 信号)。回显经 TTY 的 echo sink → master 读环 |
| `slave_read(buf,n)` | `slave_tty_.read_cooked(buf,n)` 排 committed cooked 行(~ICANON 下是裸字节) |
| `slave_write(buf,n)` | 逐字节 push 进 master 读环(程序输出),环满则 partial write |
| `master_read(buf,n)` | 排 master 读环(slave 输出 + 本地回显) |

关键决策:

- **slave 复用 `TTY`,不重写行规范**。PTY 的"终端感"全来自 slave 侧的行规范;Phase 1 的 `TTY`
  已经是纯逻辑、host 可测、带 echo sink 注入,正好被 PTY 拥有。
- **回显路由 = echo sink 接到 master 读环**。构造时 `slave_tty_.set_echo_sink(&Pty::echo_thunk, this)`,
  `echo_thunk → echo_to_master → master_push`。这样 emulator 在 master 读侧看到自己敲的字,与真终端一致。
  回显 best-effort:master 不排导致环满时丢回显字节,不阻塞输入路径。
- **EOF 与「无数据」分离**。`slave_read` 返 0 时有两种含义:cooked 空(应阻塞)vs EOF(Ctrl+D 空行)。
  批1 不在 `slave_read` 里合并,而是让调用方在返 0 时查 `slave_tty().take_eof()`——与 `TTY` 自身把
  `read_cooked` 与 `take_eof` 分开的设计一致。B3 kernel 侧据此决定返 0(EOF)还是阻塞。
- **信号字符透传**。master 写 ^C → `input_char` 返 `kSignal` 并在 TTY 内部置 `pending_signal_`;
  `Pty::take_pending_signal()` 转发 `slave_tty_.take_signal()`。B4 kernel 侧把它投给 slave 的前台进程组。
- **master 读环**手写(head/tail/full,同 `tty.cpp` 的 cooked 环),避免任何 kernel 依赖,保持纯逻辑。

## 验证

host 单测 `test/unit/test_pty.cpp`(10 例,自带 CHECK + main,链真 `pty.cpp + tty.cpp`):

1. canonical 行 round-trip(master "hi\n" → slave 读 "hi\n")
2. 本地回显(master 写后 master_read 看到 "ab\n")
3. slave 输出 → master 读("output" round-trip)
4. 信号字符(^C → pending SIGINT,slave 无行,`take_pending_signal` 消费一次)
5. raw 模式(~ICANON,逐字节立即可读)
6. 退格编辑("ab"+DEL+"d\n" → "ad\n")
7. 空行 ^D → slave_read 返 0 + `take_eof` 真(仅一次)
8. 行内 ^D → 提交该行(无 EOF、无换行)
9. 满环 partial write(slave 写满 4096 后再写返 0;排回全 'Z')
10. 双实例不串扰

**踩到的点**:第一次 `test_host` 跑 ctest 报 `pty (Not Run)`——因为 `add_cinux_test(pty ...)`
建的**可执行 target 名是 `test_pty`**(ctest 测试名才是 `pty`),而该轮 `test_host` 在 ctest 时
`test_pty` 二进制还没链出来(首编依赖时序)。直接 `cmake --build build --target test_pty` 暴露真问题
(其实是干净的,只是首跑竞态)。另外测试里忽略 `[[nodiscard]] ErrorOr` 返回值触发 `-Wunused-result`
(项目零警告,F-INFRA),全改成 `CHECK(p.master_write(...).ok())` 消警告且加断言。

- host ctest:**65/65**(含新 pty)
- run-kernel-test-all:**977/0** 两 leg(单核 + -smp 2 AP 回读 PASS)——B1 不碰 kernel,维持基线。
