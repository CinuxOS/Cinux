# F3-M2 批2 — futex 系统调用

> 2026-06-18。F3-M2 批2：`sys_futex`(202)用户态快速互斥——FUTEX_WAIT/WAKE + BITSET。256-bucket hash + 侵入式 FIFO wait queue，镜像 kernel/proc/sync.cpp 的 Semaphore block/unblock 握手。1 批，785→794（+9 测试），fresh 794/0。批5 cleartid（线程退出 futex_wake）消费此基建。

## 背景

pthread 互斥/条件变量底层是 futex：用户态先原子检查/自旋，竞争失败才陷内核睡眠。需要内核维护「按 uaddr 分桶的等待队列」，WAIT 比对 `*uaddr==val` 后阻塞，WAKE 唤醒。CinuxOS 已有 Mutex/Semaphore 的侵入式 wait queue（`Task::wait_next` + Spinlock + `Scheduler::block/unblock`），futex 直接复用同款握手。

## 设计

- **256-bucket 全局 hash 表** `g_futex_table[256]`，键 = uaddr 虚地址。`bucket_for(uaddr) = (uaddr >> 2) & 0xFF`（futex 字 4 字节对齐，移掉低位散列）。每桶 `{Spinlock lock; Task* wait_head;}`，waiter 用 `Task::wait_next` 串成 FIFO。
- **FUTEX_WAIT**：bucket lock 下直接 deref `*uaddr`（kernel 映用户地址空间，无 copy_from_user，坏指针走 PF——sys_signal convention）比对 val；≠→`-EAGAIN`（不阻塞）；==→记 `task->futex_uaddr`/`task->futex_bitset` + 入队 → 释放锁 → `Scheduler::block(self,"futex")`。WAKE 在唤醒前已把 waiter 从桶里摘掉，故唤醒后 waiter 已离队。
- **FUTEX_WAKE**：bucket lock 下遍历队列，摘下最多 `max` 个 `futex_uaddr==uaddr && (futex_bitset & wake_bitset)!=0` 的 waiter 入本地链表；锁外逐个 `unblock`（同 Semaphore::post，unblock 不持锁）。
- **BITSET**：FUTEX_WAIT/WAKE 等价 bitset=0xFFFFFFFF；`_BITSET` 变体的 val3 提供 bitset，0→EINVAL。op 先 `& ~FUTEX_PRIVATE_FLAG` 取基础 op。
- **Task 加两字段** `futex_uaddr`/`futex_bitset`（waiter 携带匹配键；0 表示未等待）。

## 关键陷阱与决策

### 1. 测试用 g_per_cpu.current（不 set_current）避免 block 挂死

futex_wait 会真阻塞（调 `Scheduler::block`）。若像 sys_signal 测试那样用 `Scheduler::set_current(task)`（设 `Scheduler::current_`），block() 里 `if(task==current_) schedule()` 会切到 idle，**测试挂死**。

解法（镜像 test_sync）：测试只设 `g_per_cpu.current = task`（**不**碰 `Scheduler::current_`）。block() 看到 `task != current_` → 标记 Blocked 后**立即返回不调度**。故 futex_wait 用 `g_per_cpu.current` 取 self（同 Mutex/Semaphore；真实调度路径里 `g_per_cpu.current == Scheduler::current()`，二者由 schedule/set_current/run_first 同步，等价）。

### 2. 全局 futex 表跨测试残留 → 每测试结尾必须 wake 清理

`g_futex_table` 是 static 全局，测试间不重置。测试用栈/静态变量作 uaddr，**连续测试的栈变量地址会复用**。若某测试阻塞了 waiter 却没唤醒，残留 waiter 留在桶里，后续测试用同地址 uaddr WAKE 时会**误唤醒**残留 waiter。

解法：每个阻塞测试结尾显式 WAKE（max=剩余数 / 匹配 bitset）清空自己的 waiter。`test_wake_count_caps`（wake max=2 留 1）和 `test_bitset_no_match`（不匹配 wake 留 1）都补了清理 wake。

### 3. 桶哈希碰撞 → WAKE 必须按 uaddr 精确匹配

不同 uaddr 可能落同桶（256 桶，地址空间大）。WAKE 遍历桶时不能「桶里有人就唤醒」，必须 `waiter->futex_uaddr == wake_uaddr`（再加 bitset 交集）精确匹配——这正是 Task 携带 `futex_uaddr` 字段的原因。

### 4. 无 timeout

FUTEX_WAIT 无限阻塞，waker 必须存在。timeout 需 PIT timer 定时唤醒（wait_queue + tick 比较），推迟（批5 或后续）。测试用「waiter 阻塞 → waker 唤醒」配对，不依赖超时。
