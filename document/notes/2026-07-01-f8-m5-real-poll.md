# F8-M5:真 poll / select(阻塞 + wait-queue + 定时唤醒)

替换 `sys_poll` 的 stub(对所有 fd 假报 POLLIN)为真阻塞 poll,新增 `sys_select(23)`,
解锁 busybox nc/wget/sh。过程中把"让 poll 正确"暴露的 **4 个阻塞点**一并修掉
(用户要求"全部本批干掉")。

## 架构(do_poll_core 统一引擎 + InodeOps::poll_events 缝)

- **统一引擎** `do_poll_core(kpollfd*, nfds, timeout_ms)`(`kernel/syscall/poll_core.cpp`)
  供 `sys_poll` / `sys_select` 共用:遍历 pollfd → 就绪检查 → 无就绪则 park → wake 后重查。
  - **无限超时**:prepare_to_wait + 注册到每个 pollable fd 的 wait queue + schedule_blocked。
  - **有限超时**:同上,再 arm 一个定时唤醒;两个唤醒源(fd 事件 / timer)谁先到都行
    (Scheduler::unblock 幂等)。
- **就绪缝** `InodeOps::poll_events(inode, waiter, &registered)` + `poll_detach_waiter`
  (kernel/fs/inode.hpp,默认"regular file 永远就绪、不注册")。各 fd 类型 override:
  Pipe(`available()`/writer 关→POLLHUP)、Socket(Udp/Tcp/Unix 查 rx 环/accept 队列)、
  console TTY(`has_cooked_data()`)。
- `sys_select(23)`:fd_set bitmap → pollfd[] → do_poll_core → revents 回填 fd_set。
  只动 fd_set 低 32 字节(= FD_TABLE_SIZE 256 位,再高的 fd 本就不存在)。

## 4 个阻塞点 + 本批修法

1. **调度器无定时唤醒**(核心)——有限超时 poll 本只能 yield 自旋烧 CPU。
   新增 `kernel/proc/timer_queue.{hpp,cpp}`:固定表 (Task*,deadline),`timer_queue_tick()`
   挂在 `Scheduler::tick`(PIT 驱动)扫过期项 unblock。poll 统一成一条真 park 路径(无自旋)。
   顺带补上 nanosleep 挂了半年的"F5-M4 timer follow-up"接口(本批未迁 nanosleep,留 follow-up)。
2. **prepare_to_wait 双入队竞态**(SMP)——另一 CPU 的 producer 在窗口里 unblock 了 poller,
   `schedule()` 又把 Ready prev 重入队,`RoundRobin::enqueue` 不去重→残留重复项。
   加 `Task::on_runq` 标志,`enqueue` 见已置位则跳过,`pick_next`/`dequeue`/`clear` 清位。
3. **fd close 不传到协议层**(无 release 钩子)——socket fd 关闭不释放协议资源(端口/FIN)。
   加 `InodeOps::release`,`FDTable::close` 关 fd 时调(锁外),`SocketOps::release → Socket::close`。
   **Pipe/FIFO 不 override**(默认 no-op)——见 follow-up DEBT-023。
4. **console poll 是 stub**——fd 0/1/2 永远报 POLLIN,靠 read 兜底阻塞。加 `TTY::has_cooked_data()`
   非破坏就绪检查 + `ConsoleTty::poll_events/poll_detach`(注册单 reader 槽,复用 read 的唤醒);
   poll_core 对"无 inode 的 fd≤2"派发到 console_tty。

## GOTCHA(本批踩的)

- **全局 tid 污染(GOTCHA#22)**:`test_poll` 用 TaskBuilder 建任务(消耗全局 `next_tid`),
  跑在 `run_scheduler_tests`(其 `test_build_basic_task` 断言 tid==1)之前→把它顶挂。
  修法:`run_poll_tests()` 挪到所有 tid-敏感测之后才调(main_test.cpp 注释标了原因)。
- **clang-format 别跑 CMakeLists.txt**(memory 铁律):.txt 被按 C 重排→CMake 解析挂。
  本批手滑把 3 个 CMakeLists 传进 clang-format 一次,`git checkout` 还原。
- **Bash 工具 CWD 不持久**:worktree 里 `cd` 进去后下一条调用 CWD 又回到主仓——导致
  clang-format 一度打到主仓文件。改用绝对路径 / `cmake --build <abs>/build`。
- **多会话 VNC :0 撞端口**(memory):QEMU 起不来(非超时)。临时 sed `qemu.cmake` :0→:7,
  跑完 `git checkout`(只动 worktree 的,别动主仓)。
- **测试 harness 单线程**:真阻塞 do_poll_core 无限超时会挂;事件 wake 用 NoRescheduleGuard
  role-play(prepare_to_wait + poll_events 注册,再 write→wake_one→unblock 直观察);
  有限超时则靠 timer_queue 真 park(不挂)。完整阻塞回路由 busybox smoke 实跑。

## 防假绿

- `test_poll_finite_timeout_parks_then_returns_zero`:空 pipe + 30ms 超时→真 park→timer 唤醒→返 0
  (证明 timer-wake 端到端,非 yield 自旋)。
- `test_poll_write_wakes_registered_poller`:注册后 write→断言 poller state→Ready(事件 wake)。
- 就绪语义精确断言(revents==POLLIN 而非只看返回码):data→POLLIN / empty→0 / writer-closed→
  POLLHUP / bad-fd→POLLNVAL / 负 fd 忽略 / 双 fd 仅就绪者置位。

## follow-up(留后续)

- **DEBT-023**:Pipe/FIFO 的 fd-close→POLLHUP/EOF 需要 **Pipe 端引用计数**(读端/写端各几个
  File 引用),last-close 才 close_reader/writer。仓促做会压垮 shell pipeline 的 EOF 语义
  (`ls|grep` 靠最后一个 writer 关 fd 才 EOF)。socket 不需要(一 fd 一 Socket)。
- **nanosleep 迁到 timer_queue**:接口已就位,把 `do_nanosleep_kernel` 的 yield 自旋换成
  arm+park,彻底消 CPU 烧。
- **poll/select fd 上限 64**(watched fds);`poll(NULL,0,-1)` 返 0(Linux 永久阻塞)——
  hobby-OS 限制,非 bug。
