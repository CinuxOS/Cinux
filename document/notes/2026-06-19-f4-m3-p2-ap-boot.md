# F4-M3 Phase 2:AP 启动(SMP 双核)

> 2026-06-19。F4-M3 Phase 2(P2-1~5)完成。**`-smp 2` 双核 AP online + idle**。
> 分支 `feat/f4-m3-trampoline`(从 main 拉,P2-1 已合 main PR#22)。基线 872/0。
> commit:b8d9cb7(qemu -smp)+ 1194345(trampoline + ap_main + BSP)+ P2-5 docs。

## 成果
F4 最难的里程碑 —— AP 启动路径从零落地。QEMU `-smp 2`:ACPI 枚举 AP → BSP INIT-SIPI-SIPI → AP 经 trampoline(16→32→64)到 ap_main → 设 GS/GDT/IDT/LAPIC → 信号 → halt。BSP 等就绪屏障后续跑到 GUI。**trampoline 首次实跑即通**(配合内联表达式修)。

## 实现(P2-2/3/4b,commit 1194345)
- **ap_trampoline.S**:@0x8000,16-bit real → 32 prot → 64 long,镜像 `boot/common/long_mode.S`。**地址用内联表达式 `(label - ap_trampoline_start + 0x8000)`** —— GAS 两遍解析同 section 符号差 = 常量,免 linker script / runtime fixup / BSP patch。debugcon 0xE9 每阶段打 1/2/3 便于定位 triple fault。
- **ap_entry_long**(higher-half 桩,内核镜像内):trampoline 带栈/kernel_cr3/cpu_id 跳来,这里切 CR3→kernel_pml4 + 设栈 + call ap_main。**不能在 trampoline(0x8000)切 CR3**:切后 0x8000 在内核 PML4 未映射 → 必须在两边都映射的内核镜像里切。
- **ap_main.cpp**:AP 设 `GS_BASE=&percpu_blocks[cpu]` + `KERNEL_GS_BASE=0`(swapgs 纪律);加载 `gdt_blocks[cpu]` + 共享 IDT(`IDT::load` 提 public);enable LAPIC(spurious 0xFF);就绪屏障(`__atomic_add_fetch`);`cli; hlt`(P2 不跑任务)。
- **boot_aps**(BSP):建临时页表(PML4/PDPT/PD identity 0-64MB + higher-half 镜像,镜像 setup_page_tables)→ memcpy trampoline 到 0x8000(direct-map)→ 注入参数(偏移 = sym - start)→ ACPI 枚举非 BSP CPU → INIT-delay-SIPI-delay-SIPI#2(仅当 AP 未 online 的兜底)→ 等就绪屏障。

## 关键 GOTCHA(执行时踩/避)
1. **GAS 宏不在语句首展开**:`$TP(label)` 内联用 GAS `.macro` 永远不展开(报 "junk after expression");`.altmacro` 也不行(只改调用语法,不改展开位置)。且本构建 GAS-direct(`#` 是注释,非 cpp,`#define` 也不可用)。**解 = 内联表达式 `(label-start+0x8000)`**,GAS 两遍解析同 section 符号差。
2. **0x8000 = bootloader stage2 加载地址**(`boot/mbr.S:22`):runtime 空闲(bootloader 启动完废弃),标准 SIPI trampoline 选址。无害但需知。
3. **CR3 切换不能在 0x8000**:trampoline 临时页表 identity 映 0x8000,但内核 PML4 不映 → 切 CR3 后下条指令 fetch @0x8000 会 fault。解 = ap_entry_long(内核镜像 higher-half,两边都映)里切。
4. **AP 栈在 direct-map 高 phys**:临时页表(只映 0-64MB + 内核镜像)不映 → AP 不能在切 CR3 前用栈。解 = ap_entry_long 切完 CR3(到内核 PML4,direct-map 全映)再设栈。
5. **`IDT::load` 原 private**:AP 需 LIDT 指向共享 IDT(不该 init 重建)。提为 public(每 CPU LIDT 是合理操作)。
6. **AP idle 用 `cli; hlt`**(非 sti;hlt):P2 AP 不碰共享调度器(`Scheduler::current_` 仍静态),interrupt handler 在 AP 上跑有并发风险 → IF=0 永久 halt 归零风险。M4 改可唤醒 + 真多核调度。
7. **双 SIPI 导致 trampoline 跑两次**(debug.log 两套 1/2/3):SIPI#1 后 AP 在 busy-wait 延迟内未 signal(GDT/IDT/LAPIC init 慢),守卫发 SIPI#2 兜底(spec 本意)。无害(一次 online)。'4' 标记来源不明但不影响。

## 验证
- `run-smp`(production -smp 2):`[SMP] INIT-SIPI-SIPI -> apic_id 1` → `[AP1] GS anchored` → `[AP1] online (apic_id=1)` → `[SMP] 1 AP(s) online` → BSP 续跑到 GUI 桌面 + worker 稳定(45s+ 无 panic)。debug.log `1 2 3`(trampoline 三阶段过)。
- `run`(production -smp 1):boot_aps no-op(`cpu_count=1`),启动到 GUI。
- `run-kernel-test`(单)+ `run-kernel-test-smp`(-smp 2 测试内核,AP 停 BIOS):872/0。
- 全量 `cmake --build build` + `test_host`:绿。

## 边界(P2 不做)
- AP 只 idle,**不跑用户任务**(留 M4 多核调度:per-CPU runq + work stealing + `Scheduler::current_` per-CPU 化)。
- **lost-wakeup**(futex/waitpid/mutex)未修(留 Phase 3);AP idle 不暴露。
- AP 不接 IPI 工作(cli;hlt)。
