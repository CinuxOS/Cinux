# F4-M5 Batch 3 — R6-Part2 lockdep 锁序图(运行时死锁检测器)— 2026-06-20

> F4-M5(同步原语)Batch 3,收官 F4-M5。把 F-INFRA I-10 的 lockdep Part1(仅「持锁跨 schedule」深度计数)升级为真锁序图死锁检测:per-CPU 持锁栈 + 全局锁序邻接图 + DFS 检环(opt-in `CINUX_LOCKDEP`,默认 OFF)。顺带修 Part1 的 SMP 缺陷。分支 `feat/f4-m4-2-3-migration`(叠 M4-2-3/R3/Batch2)。

## Part1 的 SMP 缺陷(顺带修)

`g_lockdep_held_depth`(sync.cpp)是**全局** uint32。F-INFRA I-10 做时是单核;F4-M1 后多核,两核的 `Spinlock::acquire/release` 共踩一个全局计数器 → 深度错乱(本 CPU 没持锁却读到别核的残留)。Part2 用 **per-CPU 持锁栈**取代:`g_held[kMaxCpus]` 按 `percpu()->cpu_id` 索引,每核独立。

## 设计([kernel/proc/lockdep.hpp](../../kernel/proc/lockdep.hpp) + [lockdep.cpp](../../kernel/proc/lockdep.cpp))

**数据**:
- `g_held[kMaxCpus]`:每 CPU 持锁栈(`const void* held[16]` + depth)。锁身份 = Spinlock 对象地址(静态 Spinlock 地址稳定)。
- `g_edges[256]`:全局锁序图,边 `{from,to}` = 「持 from 时取 to」= from-before-to 顺序。
- `g_edge_lock`:原始 irq-safe 自旋(test_and_set + cli),**非 Spinlock 对象**(免递归)。

**`lockdep_acquired(lock)`(Spinlock::acquire 拿到锁后调)**:
1. 递归检测:lock 已在本核持锁栈 → kpanic(recursive)。
2. 取 edge_lock(irq-safe),对栈中每个 Hi:`reaches(lock, Hi)`(图中有 lock→…→Hi 路径)→ **环** → kpanic(AB-BA)。加边 Hi→lock。
3. push 到本核持锁栈。

**`reaches(from,to)`**:有界 DFS(visited 栈,≤64 节点);溢出时停止扩展(保守:可能漏检超长环,但绝不误报)。

**`lockdep_releasing(lock)`**:LIFO pop(快路径);非 LIFO 则查找+紧凑(报 out-of-order)。

**为何 edge_lock 要 irq-safe**:Spinlock::acquire 调 lockdep_acquired → 取 edge_lock。若 edge_lock 不禁中断,普通 Guard(IRQ 开)持锁期间一个 IRQ(其 handler 取 Spinlock → lockdep_acquired → 取 edge_lock)会在同核死锁。cli 禁中断杜绝同核重入;跨核由 test_and_set 串行。

**schedule() 断言**:`g_lockdep_held_depth>0` → `lockdep_held_depth()`(per-CPU)。

## 验证

- **CINUX_LOCKDEP=ON**:`cmake -DCINUX_LOCKDEP=ON` + run-kernel-test **875/0,零误报**(无 `lockdep:` 输出/无 PANIC)—— 当前代码**无潜伏 AB-BA 锁序环**(代码质量正面信号)。
- **AB-BA spike 检出**:临时在 test_acquire_release 注入 `acquire A,B(建 A→B);release;acquire B,A(B→A 触环)` → lockdep 准确 panic:`lockdep: lock-order cycle -- acquiring 0x...AEFF while holding 0x...AEFE (AB-BA deadlock)`(两栈 Spinlock 地址相邻)。spike 验证后还原。
- **默认 OFF**:spike 还原后 run-kernel-test **875/0**,无 lockdep 输出。
- 全量 `cmake --build`(改公共头 sync.hpp + 新 lockdep.hpp)零警告;clang-format 过。

## 不做(follow-up)

- lockdep 锁「类」(Linux lockdep_init_key:同地址不同初始化视为同类)——CinuxOS 静态 Spinlock 地址稳定,按地址键控足够;动态/栈 Spinlock 不被有效跟踪(非锁序风险)。
- 长环(>64 节点)检测——保守漏检,实际不会触发。
- lockdep 统计/proc 接口(边数、检测次数)。
