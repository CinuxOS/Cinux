## GOTCHA(下一手 AI 必读,免踩坑)

### 1. 【根因】轮询 RX 在模拟器上必须读 MMIO 寄存器,否则永远收不到包

**现象**:TX 出去(filter-dump pcap 证明 SLIRP 也回了),但 `RDH=0`、`GPRC=0`,ring 里一个包都没。
ring 寄存器回读全对(RDBA/RDLEN/RCTL.EN/desc[0].buffer_addr 都正确)。

**根因**:模拟器把入站包从设备模型的 main loop 投递到 ring,而 main loop **只在 guest trap
出(MMIO 访问 / 中断 / hlt)时才跑**。描述符 status 在 DMA 内存里——读它**不 trap**——所以
纯内存轮询永远不触发投递。测试内核 CPU 中断是关的(xHCI 同款注释),没定时器/IRQ 兜底。

**铁证**:不读 MMIO → `GPRC=0 RDH=0`(包在线上却没投递);`poll_rx` 开头读 `RDH`(MMIO,trap
出 → 模型 main loop 跑 → 投递)→ `GPRC=1 RDH=1`,包到 ring。这不是 QEMU 的 bug,是轮询驱动
在模拟器上的正确交互方式(真 HW 该读也读个 head,无害)。xv6 没中招是因为它开中断 + 调度器
(频繁 trap)。**修**:`poll_rx` 开头 `(void)reg_read(RDH);`(见代码注释)。

定位法(别归因共同路径里最显眼的):GPRC(收包计数)区分「MAC 没收(filter 问题)」vs「收了
没 DMA 到 ring(delivery 问题)」。本例 GPRC=0 + ring 寄存器全对 → delivery 没发生 → 模拟器
没跑 main loop → 内存轮询不 trap。

### 2. QEMU e1000 **不模拟** RCTL.LBM 内部 loopback

bit 能写进去(RCTL 回读有 LBM_MAC),但 TX 的帧**不回环**到 RX(去了 SLIRP)。别再花时间做
loopback 自测;用 SLIRP ARP/DHCP round-trip 代替(真包,更硬)。已删 `enable_loopback` 死代码。

### 3. 测试包的目的 MAC 必须是广播(BAM)或单播(UPE);随机首字节为奇 = 组播,无 MPE 被丢

早期 loopback 测试用 `pkt[0]=(i*7+1)&0xFF=0x01` → 01:.. 是**组播**地址,UPE 只收单播 → 被丢
(RDH 不动)。广播(全 FF)走 BAM,单播走 UPE/RAL,才收得进来。

### 4. 直接跑 QEMU 别忘重生 ext2.img(否则 42 个 ext2 测试假失败)

直接 `qemu-system-x86_64 ... ext2.img` 不重生镜像,而 ext2 测试会往镜像里建文件(inode 持久
累积)→ 几次后 `alloc_inode: no free inodes available` → 42 个 ext2/fs 测试 FAIL。**这跟 e1000
无关**,是镜像陈旧。直接跑前先 `cmake --build build --target regenerate-ext2-image`;标准
`run-kernel-test` 会自动重生。

### 5. 串口抓取:`-serial stdio` + isa-debug-exit 瞬退会丢缓冲;且 `-vnc :0` 易端口冲突

直接跑 QEMU 抓串口用 `-serial file:/tmp/x.log`(别 stdio)+ `-vnc :3`(避开 :0 冲突)+ 想看
线上流量加 `-object filter-dump,id=f0,netdev=net0,file=/tmp/dump.pcap`。grep 日志带 `-a`(内核
输出有控制字符,grep 会当二进制不输出)。
