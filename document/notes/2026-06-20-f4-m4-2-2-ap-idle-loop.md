# F4-M4 M4-2-2（AP idle loop + runqueue 多核安全，限定交付）— 2026-06-20

> 原计划"AP idle loop + 真迁移 user task + -smp 2 真机"。侦察发现两个核心问题（idle_task_ 单例 + pick_next 多核双调度）合并修，但实测暴露**第三个深层问题：user task 迁移到 AP 确定性 GP**（%gs/地址损坏，swapgs 全链代码审查正确，根因需 GDB）。**用户决策分批**：M4-2-2 限定交付"per-CPU idle + runqueue 多核安全 + AP1 sti;hlt 基础设施"（不迁移），迁移 GP 划 **M4-3**。commit `3d080a0`，分支 `feat/f4-m3-trampoline`。

## 交付（M4-2-2）

1. **per-CPU idle**：`idle_task_` 单例 → `idle_tasks_[kMaxCpus]` + `idle()` 访问器（percpu cpu_id）+ `setup_ap_idle(cpu)`。BSP `idle_tasks_[0]`（idle_entry while-hlt 不变），AP `idle_tasks_[cpu]`（ap_idle_entry）。`schedule`/`exit_current` 的 idle fallback 用 `idle()`。**单核零回归**（idle()=idle_tasks_[0]）。
2. **runqueue 多核安全**：`pick_next` 删 re-enqueue（pick 后任务**移除**，运行中任务不在队列 → 两 CPU 不会双 pick 同一 Running 任务踩同栈）。`schedule()` prev 可运行时 enqueue 回队列（pick 之前）。单核 round-robin 等价（推演：唯一任务→enqueue→pick 自己→return；T1/T2→enqueue T1→pick T2→轮转）。4 个 RoundRobin 单测适配新契约（pick 移除 + 循环靠 enqueue，非 pick re-enqueue）。
3. **AP1 sti;hlt 基础设施**：ap_main 替 cli;hlt 死循环 → `cli;pause` 自旋等 `is_initialized()`（boot_aps 早于 init）→ `setup_ap_idle` → `percpu()->current=idle` → `context_switch(&dummy, &idle->ctx)` 切到 ap_idle_entry。ap_idle_entry = **sti;hlt（M4-2-2，不 schedule 迁移）**。+ reschedule IPI（M4-2-1）+ wake_idle_ap。AP1 online + 接 IPI + sti;hlt idle。
4. **lost-wakeup 重检（代码就绪，M4-3 启用）**：`has_runnable_task()`（peek 不出队）+ `RoundRobin::is_empty()`（mutable lock_）+ `SchedulingClass::is_empty()` virtual 默认。M4-3 的 schedule loop 用它 hlt 前重检。

## 发现的深层问题：user task 迁移 GP（→ M4-3）

AP1 完整 schedule（迁移 user task init/kernel_init）→ **确定性 GP**（%gs/地址损坏，GUI 不启动，run 1/2 每次 GP=71）。AP1 不迁移（sti;hlt）→ 干净（GUI + 用户程序 + AP1 online）。

**诊断三步**：
1. **诊断 1**（AP1 只 sti;hlt 不 schedule）：**干净**（AP1 online + GUI Desktop + gui_worker + kernel_init exit）。证明 AP1 idle 基础 + IPI + production 启动 OK。
2. **诊断 2**（完整 schedule + 禁 panic backtrace）：**干净**（GUI 启动）。但这是**代码布局侥幸避开 race**——禁 backtrace 改变 binary 布局 → race 时序变化 → 恰好不触发。非真干净。
3. **run 1/2**（完整 schedule + backtrace 恢复 = 生产配置）：**确定性 GP=71, GUI=0**（每次）。panic 的 backtrace 在 %gs 损坏时递归 GP（in_kernel_stack → current() → %gs → GP → panic → ...）放大表象。

**GP 现场**：dump RIP=in_kernel_stack（递归），RCX=0xC0000101（GS_BASE MSR），RAX=0xF000FF53F000FF53（非规范）。根因 %gs 被损坏（GS_BASE 非规范），之后任何 percpu()（%gs:）→ GP。

**swapgs 全链审查正确**：ISR 条件 swapgs（CPL=3 才 swap，interrupts.S:72）、syscall.S swapgs（进/出 kernel）、jump_to_usermode swapgs（usermode.S:109，kernel GS=percpu / KERNEL_GS=user）。代码层正确。**根因指令位置被 panic 递归 + 布局敏感掩盖，需 GDB -smp 2 抓第一次 GP faulting RIP**（M4-3）。临时禁 backtrace 抓 RIP 失败（布局避开 race，不触发）。

- **-smp 2 make run-smp：GP/PANIC=0**，AP1 online + kernel_init started + GUI Desktop + gui_worker，**exit 124（GUI 持续）**。AP1 sti;hlt 干净。
- 全量 `cmake --build`（host 单测 CI 盲区，改公共头 scheduler.hpp）exit 0 无 warning。
- **KVM 恢复**：用户 `sudo chmod 666 /dev/kvm` 解决 KVM 权限，run-kernel-test / make run-smp 走 KVM（快，~60-90s）。

## GOTCHA

- **#25（M4-2-2，→ M4-3）**：user task 迁移到 AP **确定性 GP**（%gs 损坏）。swapgs 全链代码正确，根因未定位（panic 递归 + 布局敏感掩盖第一次 RIP）。AP1 不迁移（sti;hlt）则干净。**M4-3 用 GDB -smp 2 抓第一次 GP faulting RIP 定位 + 修**（可能 per-CPU runqueue 不迁移，或迁移 ctx 同步）。M4-2-2 限定 AP1 sti;hlt。
- **#23（既有）**：context_switch 恢复点 current() 读 per-CPU。M4-2-2 AP1 idle 首次切换用 dummy ctx（write-only，不读），安全。
- **runqueue 多核安全**：pick_next 移除不 re-enqueue（单核 round-robin 靠 schedule enqueue prev）。多核下运行中任务不在队列避免双 pick。但 **yield 跨 CPU 迁移仍有理论 ctx 竞态**（prev enqueue 后 ctx 未 save，别的 CPU pick restore 旧 ctx）——plan 已标注，per-CPU runqueue 根治（M4-3 候选）。

## 不做（M4-3 / follow-up）

- **user task 多核迁移**（GP 根因定位 + 修）→ M4-3。GDB -smp 2 抓第一次 RIP。
- per-CPU runqueue + work stealing（避免迁移）→ 可能 M4-3 方案（根治迁移竞态）。
- per-CPU APIC timer 抢占（用户决策不做）。
