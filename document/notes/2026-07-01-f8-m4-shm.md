# F8-M4 — SysV 共享内存（shmget/shmat/shmctl/shmdt）

> 2026-07-01，worktree `wt-shm`（分支 `feat/f8-m4-shm`，从 main `5c03f85`）。F8-M4 单批。

## 背景 / 目标

F8 最后一块 IPC 经典：**两进程映射同一物理页，一个写一个读**。Linux x86_64 四个 syscall：
`shmget(29) / shmat(30) / shmctl(31) / shmdt(67)`。机制 + ring0 端到端验证；musl 用户态 demo、
SHM_LOCK/ctl 高级命令留 follow-up。

## 设计 / 决策

**分层照抄 F8-M2 的 fifo/pipe 切法**（CODING-TASTE §13a：传输/应用不混）：

- `kernel/ipc/shm.{hpp,cpp}` —— `ShmRegistry` 固定 16 槽表（`key(int) → ShmSegment`），**纯逻辑**
  （无 PMM/AddressSpace/kprintf，仿 `FifoRegistry`，可链 host 单测）。`attach/detach/mark_removal`
  在 `Spinlock` 内更新 `nattach`；仅当 `marked_for_removal && nattach==0` 时把 `phys_base` 交还
  syscall 释放——**段持有物理页，attach 仅借用**。
- `kernel/syscall/sys_shm.cpp` —— 物理页生命周期（PMM alloc/free + 按地址空间映射）。`shmget`
  走 `alloc_pages` 拿连续物理页 + 注册；`shmat` 选空闲 VMA（ASLR，同 mmap）或用户 `addr`，遍历页
  `address_space.map()` + `mapcount_inc`；`shmdt` 反向；`shmctl` 走 `do_shmctl_kernel`（IPC_STAT）
  + `copy_to_user`、IPC_RMID。

**mapcount 闭环（镜像 fork CoW，kernel/proc/fork.cpp）**——这是本批的正确性核心：

| 事件 | 每页 mapcount |
|------|--------------|
| `shmget` `alloc_pages` | **1**（段的自身引用，基线）|
| 每次 `shmat` 映进一个地址空间 | +1 |
| `shmdt` 或地址空间销毁（`free_subtree`）| −1 |
| `IPC_RMID` 显式 `free_pages` | 回收 |

`free_subtree`（`~AddressSpace`）对每个在场的用户 PTE 做 `mapcount_dec_and_test`——段自带的基线
`1` 保证它**永不因 teardown 跌到 0**，只有段的 RMID 显式 free 才回收。否则段页会在进程退出时被
`free_subtree` 误放（段还在用就进了 free pool）。fork CoW 是同款账：alloc 置 1（首主）、share 加、
unmap 减。

**`shmid = 表索引`**（0..15），不学 Linux 嵌 sequence number——教学内核 + 对齐 `FifoRegistry`
的 index-as-handle。

**`shmid_ds` 用精简内核内形状**（shm_segsz / cpid / lpid / nattch / mode），非 Linux 全 `ipc_perm`
+ 时间戳布局；musl/glibc 互操作时再拓宽。

## 陷阱

**`shmdt` 长度别取 VMA 的 `[start,end)`——取段的 `page_count`**。`vmas().insert()` 会**合并相邻
同标志 VMA**；两个背靠背的 SHM 映射（都 `Read|Write|Shared`，无 `Anonymous` 位）会并成一个 VMA，
其跨度大于单段。若按 VMA 算 unmap 长度，`shmdt` 第一个段会**顺带拆掉邻居的页**。

正解：`shmdt(addr)` 用调用方 `addr`（`shmat` 返回的段基址）→ `translate(addr)` 得 `phys_base` →
`find_by_phys` 定位段 → 取段 `page_count` 算长度 → `unmap` + `vmas().remove(addr, addr+len)`
（`remove` 戳洞，正确处理合并 VMA 的切分）。`test_shm_adjacent` 专门锁这个回归：两段贴邻、写不同
magic、`shmdt` 第一段后第二段仍读得到自己的值。

**mapcount 只 inc/dec `page_count` 页，buddy 尾页不动**。`alloc_pages(count)` 向上取整到 2 的幂
（`n_pages = 1<<order`），`mapcount=1` 置在**整块** `n_pages` 页上；映射只碰前 `count` 页。尾页
mapcount 留 1（stale 但无害：`free_pages` 不查 mapcount，下次 alloc 重置 1）。`test_shm_stat` 用
3 页段（实际 alloc 4 页）验证过。

**ring0 测 SMAP**：测内核里 SMAP 默认开，ring0 读写用户映射页要 `stac/clac` 窗（同 `test_file_mmap`）；
每个地址空间要先 `activate()`（换 CR3）再碰映射，完事 `write_cr3(kernel_pml4())` 回内核空间再让
栈上 `AddressSpace` 析构。机制证明用 `translate(virt1)==translate(virt2)`（同物理页）+ as1 写 as2
读双保险。

**worktree 子模块**：worktree 的 `third_party/Cinux-GUI` + `Cinux-Base` 默认未检出（CMake 报
`does not contain a CMakeLists.txt` / `No SOURCES given`），`git submodule update --init --recursive`
补齐（共享 `.git`，离线可成）。

## 验证

```
cmake -B build -DCINUX_MUSL_DYN_SMOKE=OFF -DCINUX_MUSL_HELLO_SMOKE=OFF -DCINUX_BUILD_TESTS=ON
cmake --build build -j$(nproc)
timeout 200 cmake --build build --target run-kernel-test-all -j$(nproc)
```

两 leg 全绿：单核 **1067 passed, 0 failed**，-smp 2 leg **ALL TESTS PASSED**。6 个 SHM 测两 leg 各
PASS：round-trip（两地址空间同物理页 + 跨 CR3 写读）、IPC_STAT、RMID 后 shmat 失败、命名 key
EXCL/重开、非法参数、相邻 detach 回归。

## Follow-up

- musl/glibc 用户态 SHM demo（需拓宽 `shmid_ds` 到 Linux 全布局）。
- `IPC_SET / IPC_INFO / SHM_LOCK` 等高级命令（现 `-ENOSYS`）。
- 权限检查（IPC_CREAT 命中既有段、shmat 的 SHM_RDONLY 与段 mode 交互）——现仅记 mode 不强制。
- SMP 下 `segment()` 快照与 `detach/mark_removal` 之间的 TOCTOU（单线程测不触发，文档化）。
