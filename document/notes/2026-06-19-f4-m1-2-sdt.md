# F4-M1 批2:RSDT/XSDT 表遍历与 find_table

> 2026-06-19。F4-M1 批 2。范围:find_table(签名)—— RSDT(32位主路径)/ XSDT 遍历 + 每表 checksum。
> 基线 = M1-1(849/0)。

## 目标
按 4 字节签名("APIC"/"FACP"/"HPET")从 RSDT/XSDT 查找 SDT 表,为 M1-3(MADT)/ M1-4(FADT/HPET)提供统一查找入口。

## 决策
- **RSDT 32位主路径**:M1-1 确认 QEMU pc 是 ACPI 1.0(rev 0,仅 RSDT);find_table 优先 XSDT(rev>=2)否则 RSDT(rev 0 实际走这条)。
- **byte-wise 读 entry**:entries 紧跟 36B SDT header(非 8 对齐),用 read_u32_le/read_u64_le 逐字节读,避免 unaligned read / -Wcast-align(零警告纪律)。
- **length 上限 64KB**:walk_root 校验 `length ∈ [sizeof(SDTHeader), 64KB]` 再 checksum,防 corrupt length 致 validate_checksum 扫 MB 级(MADT/FADT/HPET 均 < 64KB)。

## 实现
- `acpi.hpp`:`find_table(const char* signature)` 声明。
- `sdt.cpp`(新):find_table + walk_root(双 entry_size 4/8)+ sig4_matches + read_u32/u64_le + phys_sdt(direct-map)+ 每表 checksum + length sanity。
- `test_acpi.cpp`:+5 测(find APIC/FACP 非 null + signature 匹配 + length sane + unknown null)。

## 验证
- `timeout 40 run-kernel-test`:**854 passed, 0 failed**(849 + 5 find_table)。零编译警告。
- 真机:find_table("APIC")→MADT、find_table("FACP")→FADT、find_table("ZZZZ")→null。

## 产出
- `kernel/drivers/acpi/sdt.cpp`(新)、`acpi.hpp`(+find_table)、CMake(+sdt.cpp)、`test_acpi.cpp`(+5 测)。
