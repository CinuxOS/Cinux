# 2026-06-23 F5-M5 批1A — PCI find_xhci + is_xhci_device 谓词

## 背景 / 目标

Phase 1 第一步:在 PCI 层加 xHCI 控制器发现。抽纯谓词 `is_xhci_device(class/sub/prog_if)`强制 `prog_if==0x30`(避开 UHCI/OHCI/EHCI),`find_xhci` 仿 `find_ahci` 扫描循环用它。

## 变更

- pci.hpp:+ constexpr `is_xhci_device`(0x0C/0x03/0x30)+ `find_xhci` 声明。
- pci.cpp:+ `find_xhci` 实现(仿 find_ahci,匹配后 read_bars + 打印 BAR0)。
- test_pci.cpp(NEW):7 用例(xHCI 正匹配 + 拒绝 UHCI/OHCI/EHCI/USB-device/AHCI/非串行总线)。

## 关键陷阱

- **prog_if 必查**:class 0x0C/sub 0x03 不带 prog_if 会同时匹配 UHCI(0x00)/OHCI(0x10)/EHCI(0x20)。`is_xhci_device` 三段式断言锁死 0x30,host 测试覆盖四种错误 prog_if。

## 验证

- host:test_pci **7 例**全过。
- kernel:全量构建绿(find_xhci 进 pci.cpp,无调用方,dead code 无害)。
- run-kernel-test:**928/0**。
- 真发现 xHCI 的 QEMU 证明留 1C(需 qemu-xhci 设备)。
