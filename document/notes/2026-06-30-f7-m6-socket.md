## 架构决定(4-agent 设计面板核实,已读码)

**Socket = InodeOps 子类**(对齐 PTY/pipe,socket fd → File → Inode → SocketOps),`sys_read/write/ioctl/close`
**零改动**——对 fd 层 socket fd 就是一个 pipe fd。4 agent 全票通过,一致否决「给 File 加类型 tag」(那要改
每个 fd 消费者 + 未来 dup2/poll,正是 Linux `struct file` 单缝避免的反模式)。

- `socket()` 装合成 Inode(抄 `sys_pipe::do_pipe_kernel` unique_ptr→`FDTable::alloc(RDWR)`→release)。
- `accept()` 抄 `PtmxOps::open` cloning 再发一个 fd。
- `close()` 走现成 `FDTable::close`;SocketOps 析构拆连接(TCP FIN)。
- SocketOps 单一共享无状态实例;per-fd Socket* 存 `inode->fs_private`(PTY 范式)。

**协议层保持纯**:TcpModule/UdpModule(M4/M5)不动;per-socket RX 环 + 阻塞 + accept 队列放 Socket
**适配器**,挂 listener 缝(`UdpListener::on_udp` / `TcpListener::on_accept/on_data/on_close`)——回调里
**拷贝借来的帧**进环(设备 dispatch 后回收 buffer)。守住 CODING-TASTE §13(通用类不混应用语义,
xHCI/HID 反例)。

**阻塞现在就能做**(不需 timer):`prepare_to_wait/schedule_blocked/unblock`(F8 pipe 那套),从
`net_poll` kthread 上下文的 listener 回调唤醒。#DF 安全(syscall 内不 sti/hlt)。

## 关键 GOTCHA / 决定点

- **测试内核真带 e1000 + 跑生产 net_init**(`test_net.cpp:180` 调 `cinux::net::init()`),故 `g_ready=true`。
  → 工厂对 SOCK_DGRAM/STREAM 在测试内核里也造真 UdpSocket/TcpSocket(不是 stub)。B1b 的 stub 断言改宽
  (send 到达 Socket 即可,不强求 NotImplemented)。B2/B3 的 loopback echo 测试**自建栈**(function-local
  static ArpModule/IcmpModule/Ipv4Module/Udp|TcpModule/NetStack + 文件域 LoopbackDevice + route 函数指针),
  绕过工厂,镜像 `test_udp_loopback`。
- **TCP active-open TCB listener=null**(M5 局限):`TcpModule::connect` 用 `alloc(...,nullptr)`,客户端
  收不到 on_data。加 `TcpModule::set_listener(conn,l)`(additive,不破现有调用):客户端 connect 后
  `set_listener(*this)`,accepted child 在 accept 里 `set_listener(*child)` 重绑 TCB listener 从 parent→child。
  两种 Connected 角色(客户端 / accepted child)都是自己 TCB 的 listener,on_data 直达。
- **TCP 是字节流**(非数据报):用 `cinux::lib::RingBuffer<uint8_t,4096>`(像 pipe),on_data push_batch,
  recv pop_batch;UDP 用定长 Datagram 环(每包 {src,port,len,data[1472]})。
- **borrowed frame 必须拷贝**:`on_udp`/`on_data` 的 FrameView 借自设备 buffer,dispatch 返回后回收;
  回调里 copy 进环(udp.hpp/tcp.hpp 注释明示)。
- **生产 add_l4 容量**:`Ipv4Module::kMaxL4=4`(ICMP ctor 自动占 1),UDP+TCP=3 槽放得下。
- **解耦门** `check_net_decoupling.sh`:`kernel/net/` 不能 include driver/dma/irq。socket/udp_socket/
  tcp_socket 放 kernel/net/(只用 net 模块 + scheduler 等待队列 + RingBuffer,合法);实例化 + add_l4 +
  create_socket 工厂 + dev_for 路由放 net_init.cpp(drivers/net/,唯一看到双方处)。
- **sockaddr_in port 是网络序**:musl 大端铺;syscall handler `byte_swap16`(ntohs/htons)转主机序再给 Socket。
- **syscall 槽位核实**:`syscall_nums.hpp` 39(getpid)→56(clone)之间 40-55 全空,socket=41/connect=42/
  accept=43/sendto=44/recvfrom=45/bind=49/listen=50 直接用 Linux x86_64 号,无撞号。
- **阻塞 vs sti/hlt(致命)**:socket recv/accept 在 syscall 上下文绝不 sti/hlt(→ #DF,memory
  sys-ping-df-sti-in-syscall);走 prepare_to_wait/schedule_blocked,由 net_poll kthread 唤醒(同 F8 pipe)。
- **Error 枚举无 socket 专用变体**(ENOTCONN/EADDRINUSE 在 Cinux-Base):用现有 Error(InvalidArgument/
  AlreadyExists/Busy)近似,to_errno 翻成 -EINVAL/-EEXIST/-EBUSY。B4 musl 若对 errno 敏感再精化。

## 范围栅栏(诚实,deferred)

- **TCP 重传 / RTO / 窗口 / 拥塞**:TcpModule 一个没有(M5 范围),需内核 timer=HPET 周期中断(F5-M4
  follow-up,硬前置非 M6 活)。loopback 零丢包 → echo 稳;SLIRP 真网无重传可能丢。
- TIME_WAIT / ISN 随机化 / 乱序重组 / IPv6 / AF_UNIX(F8)/ sendmsg cmsg / epoll(F8)/ 第二 NIC /
  中断驱动 RX(仍 net_poll 轮询)/ setsockopt(留 stub)/ getsockname/getpeername/accept4 精化。
