# F5-M6 批a — e1000 NIC 驱动:PCI 检测 + 复位 + EEPROM MAC

> 里程碑:F5-M6(e1000 网卡驱动,F7 网络协议栈的地基)。
> 分支 `worktree-f5-m6-e1000`(worktree,从干净 main `8be32d4` 拉,与 feat/f9-security 上的 F9 并行,零依赖)。
> 范围决策(用户 2026-06-25):「先把驱动打通」——跳过立项 docs,直接驱动;批a 检测+MAC,批b 收一帧。

## 背景

ROADMAP 明写「F5(网卡)→阻塞整个网络栈」。F5 只做到 M5(xHCI),**M6 e1000 / M7 VirtIO
Net 未做**,所以 F7(网络协议栈 M1 以太网 … M6 Socket)动不了。「推进网络」第一公里 = 立
e1000 驱动(它既是 F5-M6,也是将来 F7-M1 以太网层的地基)。

整条 HW 跑道已被 xHCI/AHCI 实战验过(PCI 枚举 / DmaPool / MSI-X / IDT ISR / polled fallback
全就绪),e1000 是现有套路的自然延伸,模板 = `xhci_controller.cpp`。**polling 优先**:QEMU
under nested-KVM 不可靠锁存 MSI-X(xHCI 同款理由),批a/批b 不碰中断。

## 设计点

- **MMIO 槽位 +0x50000 / 8 页(32KB)**:grep 确认 KMEM_MMIO 已占 AHCI+0x0 / LAPIC+0x10000 /
  IOAPIC+0x11000 / xHCI+0x20000 / MSI-X Table+0x40000 / PBA+0x41000。e1000 BAR0 32-bit MMIO,用到
  ~0x5404(RAL0)需 ≥6 页,取 8 页留余量;+0x50000 零碰撞、远低于 2MB 上限。**这是本仓致命
  GOTCHA**(xHCI/MSI-X BAR 碰撞曾崩),亲自 grep 核过,还揪出初版「4 页」不够覆盖 RAL0/MTA 的错。
- **EERD 读 EEPROM**:QEMU 8254x 地址在 bits 11:8、数据在 bits 31:16、START=bit0、DONE=bit4。
  有界轮询 DONE(QEMU 即时,防挂);3 个 word 拼 6 字节 MAC。
- **复位非致命**:CTRL.RST 有界轮询自清;不清只告警继续(MAC 读 + 链路才是批a 交付,QEMU
  即时复位;对齐用户「可用可调试 > 严格」偏好)。
- **test 门禁用 `#ifdef CINUX_NET`**:对齐 main_test.cpp 既有的 `#ifdef CINUX_GUI` 模式(CMake
  只在 CINUX_NET on 时编 test_e1000.cpp + 守卫调用),CINUX_NET=OFF 也干净链接 —— 比 xHCI 未
  守卫的 test_xhci(预存 test 盲区)更稳。
- **MSI not MSI-X**:82540em 只支持 MSI(cap 0x05),中断批不能抄 MsixController —— 显式延后。

## 验证矩阵

- `run-kernel-test`(带 `-device e1000`):**932/0** ALL PASSED

  链路/BAR0 映射全通)
- 全量构建(改了 pci.hpp 公共头):绿,host 单测链通

## 边界 / 延后

- **批b**:RX 描述符环(legacy 16B)+ RCTL 编程 + poll_rx 收一帧(QEMU user-net 广播 DHCP/ARP)。
  RAL0/RAH0/MTA 编 MAC、net::poll_rx() boot 接口 + net_stub.cpp 在批b 引入。
- **TX 路径**:延后。
- **中断替代 polling**:延后(82540em=MSI only,需新 MSI controller 或 legacy INTx→vector 0x41;
  QEMU nested-KVM MSI 也不稳,留 polling fallback)。
- **netdev 抽象 + 交接 F7 协议栈**:留 F7。
- **生产 main.cpp 接 net::init()**:批a 只走 test 路径验证;main.cpp 接线留批b/后续。
