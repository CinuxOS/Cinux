# F6-M4 批1:TmpFs 内存文件系统(动态 inode + create/mkdir/unlink)

> 2026-07-01。分支 `feat/f12-gcc-selfhost`(从干净 main `5c03f85`,**GCC 自举主线**主 checkout 串行,不拉 worktree)。commit `6656096`,**未 push**。
> GCC 自举第一刀:给内核一个**可写的内存型虚拟 FS**,作为 `/tmp` 底座——GCC/cc1/as/ld 把中间 `*.o`/`*.s` 写进 `/tmp`。CinuxOS 此前**没有可写内存 FS**(DevFS/ProcFS 只读/定长,ext2 磁盘后端)。本批只做 TmpFs 类 + 机制测;boot 自动挂 `/tmp` + sys_mount/umount2 是**批2**。

## 设计(决定打法,已读码核实)

- **主对象 `TmpNode`**([tmpfs.cpp](../../kernel/fs/tmpfs/tmpfs.cpp)):堆分配,**内嵌 `Inode`**,`inode.fs_private` 指回 node。InodeOps 经 `static_cast<TmpNode*>(inode->fs_private)` 取回 node——延用 DevFs `fs_private=this`([devfs.cpp:150](../../kernel/fs/devfs/devfs.cpp#L150))的招,扩展到 per-node 可变内容。内嵌 Inode 地址稳定直到 node 释放(close 只删 File 不删 Inode,见 [file.cpp](../../kernel/fs/file.cpp))。
- **目录走单向兄弟链表**(`first_child` + `next_sibling`)——**无上限**,区别于 DevFS 定长表。readdir 走链表到第 N 项(idx 0/1=`.`/`..`,idx≥2→第 (idx-2) 个 child);create 头插;unlink 摘链(带 prev 指针)。
- **文件内容**:`uint8_t* data` + `capacity`/`size`。write 越界则扩容(`new uint8_t[newcap]`→memcpy 旧→`delete[]` 旧;capacity 按 4KB 对齐摊销),offset 处写入,`offset>size` 时 gap **零填**(防 stale 堆字节泄漏)。
- **锁**:单 per-FS `Spinlock` 守所有树变更 + 内容 I/O。低竞争(/tmp 专用),仿 [vfs_mount.cpp](../../kernel/fs/vfs_mount.cpp) 的单一 `g_mount_lock`,无 per-node 锁(避父/子 AB-BA)。
- **lookup 多段 walk**:仿 [ext2_init.cpp:268](../../kernel/fs/ext2/ext2_init.cpp#L268)——剥前导 `/`,逐段在 child 链里按名找;非末尾段不是目录 / 缺段 → NotFound。syscall 层 `split_pathname` 会把 `sub/file` 拆给 lookup,故 tmpfs lookup **必须自己 walk**(DevFs/ProcFS 扁平结构没这需求)。
- **page-cache 路由(已核实)**:`is_page_cacheable()` 保持默认 false([inode.cpp:85](../../kernel/fs/inode.cpp#L85))→ `do_read_kernel` 走直连 `ops->read`([sys_read.cpp:48](../../kernel/syscall/sys_read.cpp#L48));`do_write_kernel` **无 gate、恒走 `ops->write`**([sys_write.cpp:50](../../kernel/syscall/sys_write.cpp#L50))。tmpfs 内容永远不进磁盘 PageCache。

## 机制测试(防假绿,9 测,直驱 InodeOps)

栈局部 `TmpFs` + [big_kernel_test.h](../../kernel/test/big_kernel_test.h) helpers(`create_or_null`/`write_or_neg1`/`read_or_neg1`/`readdir_or_neg1`/`unlink_rc`/`lookup_or_null`)——不碰全局 mount 表/fd 表(测试内核跨 section 共享,碰了会污染后续测)。

- mount + root(Directory)+ 幂等;create→write "hello tmpfs"→read 逐字节精确 + EOF 返 0。
- append(offset 3 续写)+ offset 读;**grow 过 4KB 边界**:写 50B 'A' @0、50B 'B' @4090(need 4140>4096 → 扩到 8192)→ 验 'B' 块 / 'A' 块 / gap[50,100) 全零。
- mkdir→readdir 列出 → 子目录 create → lookup `sub/inner`(多段 walk,命中同一稳定 inode)→ readdir sub 列 "inner"。
- stat:file(`S_IFREG|0644`+size+nlink1+ino≠0)、dir(`S_IFDIR|0755`+nlink2)。
- unlink file→lookup NotFound;unlink 空目录 ok;unlink 非空目录→错(删 child 后再删 ok)。
- 负测:lookup 不存在→null;create 同名重复→null(AlreadyExists);穿文件 lookup `x/anything`→null。

## GOTCHA

- **`class`/`struct` 前向声明不匹配(-Wmismatched-tags)**:头里 `class TmpNode;`、cpp 里匿名校 `struct TmpNode` → 编译红。根因:匿名 ns 的 `TmpNode` 是**另一个类型**(cinux::fs::{anonymous}::TmpNode),与头的 cinux::fs::TmpNode 不沾边 → `root_` 指向不完整类型。正解:`struct TmpNode` 定义在 **cinux::fs(非匿名 ns)**,头前向声明也用 `struct`。
- **unlink 非空目录的 errno**:Error 枚举(Cinux-Base 子模块)无 `DirectoryNotEmpty`,故返 `Error::IOError`→syscall 边界映射 kEio(非 kEnotempty)。契约(op 失败即可)满足;真 ENOTEMPTY 需子模块加枚举值或 to_errno 特判,留 follow-up。
- **inode 生命周期(诚实简化)**:unlink 立即释放 node。Linux 用 inode 引用计数兜底"unlink 时仍有 open fd";CinuxOS 无 inode refcount,依赖 close-before-unlink(GCC 临时文件模式正是如此)。同族于 FDTable "每次 dup 新 File" 简化(F-ECO 批4),留 follow-up。

## follow-up(留批2 / 后续)
