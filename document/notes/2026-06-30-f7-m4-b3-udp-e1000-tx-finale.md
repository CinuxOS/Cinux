## 做了什么

[test_net.cpp](../../kernel/test/test_net.cpp) 加 `test_udp_e1000_tx`:在真 e1000 +
QEMU SLIRP 上发 UDP 到 10.0.2.2。镜像 test_ping_e1000 的握手(ARP miss→request out→
poll sti/hlt→SLIRP ARP reply→cache hit),断言:
- `arp.lookup(kSlirpGateway, gw_mac)` 真——SLIRP 答了 ARP,真 e1000 双向通;
- `sent_ok`——`udp.send` 经 ipv4.send 接受了 proto=17 包。

**SLIRP 无 UDP echo 服务,不期望 reply**——这是 TX-only smoke,证 UDP 蹭通 ping 已验的
e1000 L2/L3 TX 路径(不引入新失败模式)。沿用 ping 的 LAPIC timer + sti/hlt 时序(绝不写
trap-loop 泵)。

## M4 收官

F7-M4 UDP 协议层三批全绿:

| 批 | 内容 | 验证 |
|---|---|---|
| 1 | UDP 协议核心 + L4Handler 表(ICMP 迁入)+ host 单测 | ctest 63/63 + 两 leg 967/0 |
| 2 | loopback UDP round-trip 内核测 | 两 leg 968/0 |
| 3 | e1000 UDP TX smoke + docs | 两 leg 969/0 |

- **协议层**:封装(UDP 头 + 伪首部校验和连续缓冲区法)/ 解析(IPv4 proto=17 经 L4 表
  分派)/ 端口多路复用(bind/unbind + UdpListener)。
- **架构**:Ipv4Module 的 L4 分派从硬编码 ICMP 升级成 proto→handler 表(还掉 TODO),
  ICMP 自动迁入(ctor 注册),UDP `add_l4(17)`。单一分派机制,加 TCP 不疼。
- 范围栅栏守住:socket API(sys_socket/bind/recvfrom)留 F7-M6、TCP 留 F7-M5、不进生产
  `net::init()`(无消费者)。
