# F2-M3: brk Syscall + 用户态堆

> 日期 2026-06-17 · 里程碑 F2-M3 · 分支 `feat/f2-m3-brk`
> 实现 `sys_brk`（Linux 12），用户态 malloc 的底层接口。

## 背景

mmap（M2）给用户程序离散映射；brk 给传统连续堆（sbrk/malloc 底层）。M3 在 M1 VMA + M2 mmap 懒分配基础上实现 brk。

## 设计

- **懒 brk**（关键决策）：`sys_brk` 只调 `brk_current`（边界检查），**不 map/unmap 页**。Heap VMA `[brk_initial, USER_BRK_MAX)` 在 execve 建（覆盖整个堆窗），堆增长访问 PF 时走 demand paging（和 mmap 懒分配统一，复用 M1 批4 诊断）。
- Task 加 `brk_current`/`brk_initial`/`brk_max`。
- `sys_brk` 语义（Linux）：addr==0 返当前；越界（`<brk_initial` 或 `>brk_max`）返当前（忽略）；否则 `brk_current=addr` 返 addr。**不返 errno**（返地址）。
- execve：PT_LOAD 循环记录 `max_seg_end`，设 `brk_initial`=页对齐段末尾，`brk_max=USER_BRK_MAX`，建 Heap VMA。

## 关键决策

1. **懒 vs eager**：todo T2 说 brk 扩展 map 页 + 收缩 unmap。选懒（调 brk_current 不碰页），与 M2 mmap 懒分配统一，简单 + 复用 demand paging。代价：堆页 PF 时才映射（和 mmap 一致）。
2. **brk_initial = ELF 段末尾**（非固定 USER_BRK_BASE）：PT_LOAD 循环跟踪 `max_seg_end`，brk 紧接 ELF 镜像（标准 Linux 行为）。
3. **Heap VMA 覆盖整个窗** `[brk_initial, USER_BRK_MAX)`：brk 扩展不需改 VMA（brk_current 在其中），简化。

## 陷阱

- **syscall handler 6 参**：`sys_brk(uint64_t×6)` 匹配 SyscallFn（M2 教训，即使只用 1 参）。
- **execve 路径不被 run-kernel-test 覆盖**：brk_initial/Heap VMA 靠实机冒烟（启动到 GUI 不炸）+ sys_brk 单测（set_current + brk 字段）。
- **brk 不返 errno**：返地址（Linux），与 mmap/munmap（返 -errno）不同。

## 验证

- run-kernel-test：721→722（+1：brk query/extend/bounds），全绿。
- 全量 build（含 host）：CI 对等（Task brk 字段公共头）。
- 实机冒烟（timeout make run）：启动到 GUI worker 无 panic/halt。
