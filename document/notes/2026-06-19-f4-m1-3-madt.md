# F4-M1 批3:MADT 解析 → ACPIInfo

> 2026-06-19。F4-M1 批 3。范围:解析 MADT,提取 LAPIC 基址 / enabled CPU APIC ID / IOAPIC 地址+gsi / IRQ override → ACPIInfo。
> 基线 = M1-2(854/0)。这是 M1 的核心批次:ACPIInfo 是 M2(APIC init)的直接输入。

## 目标
把 MADT("APIC" 表)的变长 ICS(Interrupt Controller Structure)列表解码成 M2 可直接消费的 `ACPIInfo`:Local APIC MMIO 基址、enabled CPU 的 APIC ID 列表、第一个 I/O APIC 基址+gsi_base、PC-AT-compat 标志、ISA IRQ→GSI override。

## 决策
- **packed struct + reinterpret_cast 读 ICS**:entry 变长且起点任意对齐(紧跟 44B MADTHeader,各 entry length 8/12/10 累加偏移),用 `[[gnu::packed]]` struct + reinterpret_cast,编译器对 packed struct 生成对齐安全访问(避免手写 byte-wise,既正确又零警告)。
- **online-capable 当 enabled 处理**:Processor Local APIC flags bit 0=enabled / bit 1=online-capable(ACPI 6.3+),两者都计入 cpu_apic_ids(未来 AP 启动需要完整 APIC ID 集)。
- **只记第一个 IOAPIC**:QEMU pc 单 IOAPIC,ACPIInfo 存首个;多 IOAPIC 留 M3+(届时 gsi_base 区间)。
- **IRQ override 存进 ACPIInfo**(不重解析):M2-3 PIC→APIC 切换需 ISA IRQ→GSI 映射(QEMU 通常 IRQ0→GSI2 override),一次解析全提取,M2-3 直接读。

## 实现
- `acpi.hpp`:MADTHeader / ICSHeader / ProcessorLocalAPICEntry / IOAPICEntry / InterruptSourceOverrideEntry(packed)+ ACPIInfo(含嵌套 IrqOverride[16])+ parse_madt 声明。
- `madt.cpp`(新):parse_madt —— null/length 守卫 → 提 LAPIC 基址 + pcat_compat → 遍历 ICS(type switch:LocalAPIC 收 enabled CPU / IOAPIC 记首个 / Override 收映射 / default skip NMI/x2APIC)→ malformed entry(length 越界)break。
- `test_acpi.cpp`:+5 测(CPU≥1 / LAPIC=0xFEE00000 / IOAPIC=0xFEC00000 / pcat_compat / null 空)。

## 验证
- `timeout 40 run-kernel-test`:**859 passed, 0 failed**(854 + 5 MADT)。零编译警告。
- 真机:parse_madt 正确提取 QEMU pc 标准拓扑(LAPIC 0xFEE00000 / IOAPIC 0xFEC00000 / BSP 在列 / 8259 兼容)。

## 产出
- `kernel/drivers/acpi/madt.cpp`(新)、`acpi.hpp`(+MADT 结构/ACPIInfo/parse_madt)、CMake(+madt.cpp)、`test_acpi.cpp`(+5 测)。
