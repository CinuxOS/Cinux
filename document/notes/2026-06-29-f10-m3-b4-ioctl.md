# F10-M3 批4 — ioctl 实命令(TCGETS/TCSETS/TIOCGWINSZ)

> 2026-06-29。接 F10-M3 批1-3(TTY 行规范核心 + keyboard 接入 + stdin 阻塞读,**已合 main** 经 SMAP saga PR#44)。批4 把 `sys_ioctl` 从 `-ENOTTY` 桩接成真终端 ioctl,解锁 musl/glibc 行缓冲。

## 背景

批1-3 已合 main(`53ef726`/`ced0066`/`35cb419`,经 `fix/smap_bug_fix` → PR#44),但 PLAN 批表仍标 ⏳(文档债,本次顺带修正)。`sys_ioctl` 仍是 F10-M1 批4 留的桩——任何 request 返 `-ENOTTY`,注释自承"enough for musl's stdio"(musl 把 -ENOTTY 当非 tty → 行缓冲凑合用)。但 glibc + 真实 isatty 路径要 TIOCGWINSZ 返尺寸;TCGETS/TCSETS 是 termios 控制(raw mode / 信号字符配置的前置)。批4 把这些接通。

## 范围

- **TCGETS / TCSETS**:读写 console TTY 的 termios(`console_tty().termios()` / `set_termios()`)。
- **TIOCGWINSZ**:返窗口尺寸(80×25)。
- **fd 0/1/2**(stdin/stdout/stderr,都 back onto console TTY)才答;其他 fd → `-ENOTTY`;未知 cmd → `-ENOTTY`。
- **TIOCGPGRP/TIOCSPGRP + Ctrl+C/^Z 信号字符留批5**(前台进程组逻辑内聚:`console_tty_input` 现只处理 `kLineReady/kEof` 唤醒 reader,**未处理 `kSignal`** → Ctrl+C 当前不投 SIGINT)。

## 实现

- **`kernel/drivers/tty/tty.hpp`**:加 Linux x86_64 ioctl UAPI——cmd 常量(`kTcgets`=0x5401 / `kTcsets`=0x5402 / `kTiocgwinsz`=0x5413 / `kTiocgpgrp`=0x540F / `kTiocspgrp`=0x5410,`<asm-generic/ioctls.h>` magic 'T')+ `struct Winsize{ws_row,ws_col,ws_xpixel,ws_ypixel}`(`<asm-generic/termios.h>`)。与既有 Termios UAPI 同族同头。
- **`kernel/syscall/sys_ioctl.cpp`**:`switch(request)` 分派;每命令走 `cinux::user::copy_to_user`/`copy_from_user`(F-EXTABLE extable 注解的 rep movsb 访问器,SMAP 安全),坏用户指针 → `-EFAULT` 不 panic。winsize 固定 `kConsoleWinsize{25,80,0,0}`。
- **`kernel/test/test_syscall.cpp`**:把单个 `test_sys_ioctl_returns_enotty` 拆三——非 tty fd(99)→ `-ENOTTY`、未知 cmd → `-ENOTTY`、TCGETS 未映射址(`0x7000000000`)→ `-EFAULT`(证走 extable accessor,非裸解引用)。

## 验证

- `run-kernel-test-all` 两 leg:**964 passed / 0 failed** + F-VERIFY M3-2 AP wake readback PASS(cr4/lstar/efer)。
- `ctest` host:**62/62**(tty host 单测 9 cases 仍绿)。
- **关 smoke 验证**:本地 `build/musl/hello` 未 build → `CINUX_MUSL_HELLO_SMOKE`(默认 ON)的 hello_smoke 20-iter poll 循环撑满 timeout 挂死;`cmake -B build -DCINUX_MUSL_HELLO_SMOKE=OFF` 后正常。CI 有 /hello 不受影响。

## GOTCHA

1. **TCGETS 负测址避 identity-map**:用 `0x7000000000`(access_ok 过但 copy 时 #PF,extable fixup → false → -EFAULT),与 F-EXTABLE B4 accessor 负测同址。低址如 0x1000 可能被 identity-map 覆盖 → copy 不 fault → 返 0(假成功),测试误判。
2. **smoke 默认 ON + 本地无 /hello → run-kernel-test 挂死**:`musl_hello_smoke_entry`(main_test.cpp:149)fork+execve `/hello` 20 iter,每 iter 最坏 50M spins waitpid poll;`/hello` 缺失 → 全 FAIL 撑满 120s timeout。本地验证关 smoke;根因是 `/hello` 非 CMake target(tools/musl/build-hello.sh 手动产)。
3. **Console 是 main.cpp 局部变量**(`Console console;` @ main.cpp:186,注册为 kprintf sink),syscall 层不可达 → winsize 取不到真实 framebuffer 几何,固定 80×25。真实几何待全局化 Console(或 DevFS 给 fd 真设备身份)。
4. **批1-3 PLAN 批表滞后**:PLAN 把批1-3 标 ⏳,实际已合 main(经 SMAP saga)。本次顺带修正 PLAN 批表 + 现状描述。

## Follow-up

- **批5 信号生成**:`console_tty_input` 处理 `InputResult::kSignal` → `TtySignal` 映射 `killpg(foreground_pgid, SIGINT/SIGQUIT/SIGTSTP)`;TIOCGPGRP/TIOCSPGRP 用 `Task::controlling_tty` + 前台 pgid。Ctrl+C 当前无效(用户可感知缺口)。
- **winsize 真实几何**:全局化 Console 或 DevFS。
- **TCGETS 端到端**:musl/glibc 实跑确认 termios 布局 round-trip(批1 host 单测已证 termios 逻辑;布局对齐 Linux UAPI 注释已核实)。
