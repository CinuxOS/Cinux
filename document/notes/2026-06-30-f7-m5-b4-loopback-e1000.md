# F7-M5 批4 — 内核 loopback TCP round-trip + e1000 TX smoke

> F7-M5 第四批。commit `cf0db3e`。把前 3 批的 TcpModule 接到**内核** in-kernel 测试栈（test-only，
> 不进 production `net_init.cpp`）。

## 背景 / 目标

批1-3 在 host 单测证明 TCP 状态机（wire / 校验和 / 握手 / 数据 / 挥手）逻辑对。批4 在**真内核 QEMU** 上
端到端复验：loopback 确定性跑完整生命周期（握手+数据+挥手），e1000+SLIRP 证明 TCP 段走通真 TX 路径（ARP resolve）。

## 设计

### test_tcp_loopback（确定性，无 SLIRP）

一个 `TcpModule` 在 `LoopbackDevice` 上跑**双端**（client 1234→7777 + server 7777→1234 同居连接表）。
`NetStack::poll()` 的 budget loop（64）让**单次 poll 排干整个三次握手**：SYN→（dispatch 期间 send_l3 入队）
SYN-ACK→（下一 budget 轮）ACK→server ESTABLISHED + on_accept。

数据 + 挥手分 poll 驱动（stateful，不能预排队）：
- **握手**：`connect` → 1 个 poll（SYN/SYN-ACK/ACK 全排干）→ 双方 ESTABLISHED + on_accept。
- **数据**：`send` → 1 个 poll（数据段→server on_data + ACK→client no-op）。
- **挥手**（2 poll）：`close(client)` → poll 1（client FIN→server ACK+on_close+CloseWait；client→FinWait2）；
  test 观察到 on_close 后**显式** `close(server)` → poll 2（server FIN→client ACK+Closed；server LastAck→Closed）。

**避 on_close 重入 close()**：曾考虑 server listener 在 on_close 里自动 close()（单 poll 完成挥手），但有两坑：
① handle 在 on_close **之后**才设 state=CloseWait，重入时 close() 看到 Established 会错走 FinWait1；
② ACK 与 FIN 的入队顺序决定 client 先收谁，FinWait1 不处理 FIN 会丢。正解：**不在 on_close 重入**，
test 显式分 poll 驱动 server.close()——干净、无重入、状态可逐步断言。

### test_tcp_e1000_tx（真 NIC TX smoke）

镜像 `test_udp_e1000_tx`。TCP `connect` 是一次性（同 4-tuple 重复被拒），不能像 UDP 那样循环 send 重试 ARP。
解法：
1. **1st connect**（→ 10.0.2.2:7777）触发 ARP request（ipv4.send ARP miss → 发 ARP + SYN 延后，无重传故可能未发）。
2. **sti/hlt + poll 循环**排干 SLIRP 的 ARP reply（同 ping/UDP 时序，**非 trap-loop 泵**）。
3. ARP 缓存后 **2nd connect**（→ :7778，不同远端口 = 新 4-tuple）——ARP hit → SYN 经 `ipv4.send(proto 6)` →
   e1000 真发出。断言 `arp.lookup(gw)` + 2nd connect ok。

SLIRP 无 TCP echo，不期望 reply（TX-only，诚实同 UDP）。

## 陷阱

- **LoopbackDevice 单 poll 多包靠 budget loop**：握手 3 包全靠 dispatch 期间 send_l3 入队 + 下一 budget 轮排干；
  若某步不生成下一段会早停——本里程碑每步都生成，全通。
- **e1000 connect 一次性**：同 4-tuple 二次 connect 返 AlreadyExists。用不同远端口绕过（2nd connect 证真 TX）。
- **VNC :0 并发会话互撞**：本机多 worktree 会话全用硬编码 `-vnc :0`；本 worktree 验证临时切 `-vnc :5`、
  跑完 `git checkout -- cmake/qemu.cmake` 还原（非代码改动，不入 diff）。
- **CINUX_BUILD_TESTS 默认 OFF**：host 单测要 `-DCINUX_BUILD_TESTS=ON`；内核 run-kernel-test-all 不受影响。
